# Архитектура проекта

## Формат запуска

Приложение запускается с одного файла `index.html`. Внутри него три логических шага:

1. Подготовка данных.
2. Выбор участников.
3. Проверка и выгрузка.

Это сделано специально, чтобы пользователь не переходил по нескольким HTML-файлам и мог открыть проект двойным щелчком.

## Структура папок

- `index.html` — единственная точка входа.
- `assets/vendor/` — локальные библиотеки `SheetJS` и `ExcelJS`.
- `styles/` — стили интерфейса.
- `scripts/core/` — состояние, конфигурация, утилиты, хранение.
- `scripts/data/` — нормализация, маппинг, расчеты.
- `scripts/excel/` — чтение Excel и выгрузка итоговой заявки.
- `scripts/ui/` — логика экранов внутри `index.html`.
- `data/` — пример конфигурации сопоставления.
- `docs/` — пользовательская и техническая документация.

## Библиотеки

### `SheetJS`

Используется для чтения входных `.xlsx` и `.xls`. Это важно, потому что файл АСУ сейчас имеет старый формат `.xls`.

### `ExcelJS`

Используется для выгрузки итогового `.xlsx` на базе встроенного шаблона `ЗаявкаШколы71_ГТО.xlsx`. Библиотека позволяет менять нужные ячейки и сохранять большую часть форматирования.

## Хранение состояния

Используются три уровня:

1. `localStorage` — автоматическое хранение на текущем устройстве.
2. JSON-экспорт — перенос проекта на другой компьютер.
3. File System Access API — сохранение JSON в выбранную папку.

Проектный JSON содержит не только настройки, но и копии загруженных файлов в сериализованном виде. Благодаря этому проект можно перенести на флешке и открыть без повторной загрузки исходников.
