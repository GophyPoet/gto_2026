# Поток данных и маппинг

## Источники

Приложение собирает итоговую заявку из четырех источников:

1. Школьный Excel-файл.
2. Файл АСУ РСО.
3. Шаблон заявки.
4. Поля, которые вводит пользователь.

## Реальное распределение полей

### Школьный файл

Используется для:

- ФИО;
- УИН;
- класса;
- списка учеников по листам.

### АСУ РСО

Используется для:

- пола;
- даты рождения;
- серии документа;
- номера документа;
- адреса проживания;
- резервного класса.

### Шаблон

Используется для:

- структуры итоговой книги;
- расположения строк заявки;
- расположения директора и даты;
- таблицы ступеней по возрасту.

### Поля пользователя

Используются для:

- названия школы;
- ФИО директора;
- даты предоставления заявки;
- даты прохождения ГТО.

## Логика маппинга

Школьный файл разбирается по смыслу заголовков `ФИО`, `УИН`, `№ п/п`.

Файл АСУ РСО разбирается по двухстрочному заголовку. Приложение объединяет строки в названия вроде `Адрес проживания | Дом`, поэтому работает не по буквам колонок, а по смыслу.

## Формирование полей

- возраст считается на дату прохождения ГТО;
- ступень определяется по листу `ступень-возраст`;
- номер документа собирается как `серия + № + номер`;
- адрес собирается из частей адреса проживания;
- если данные не найдены, ставится `-`, добавляется предупреждение и подсветка в Excel.
